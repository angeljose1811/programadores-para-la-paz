Cuando se empiezan a hacer proyectos de software, llega un punto en el que son tan complejos que es fácil perder el rumbo de los cambios que has realizado y que archivos has modificado, esto es incluso peor cuando estas trabajando con mas de una persona, donde se producen mas cambios en el proyecto y cada uno tiene ideas distintas que pueden llevar a que nunca te enteres acerca de que archivos se modifican. Es debido a esta situación que existen herramientas como Git que junto con GitHub permiten versionar tu código, es decir, tener un historial de los diferentes cambios que se producen en el proyecto para poder identificar que cambios se hicieron, quien lo hizo, cuando lo hicieron, y con que propósito; y en caso de que sea necesario es posible revertir el código hecho para volver a una versión anterior, e incluso es posible crear diferentes ramas del proyecto para testear cambios que una vez estés completamente seguro que pueden añadirse al proyecto puedes hacer un push que actualice el repositorio principal. Además, Git es una herramienta fundamental para las empresas ya que facilita el trabajo de diferentes personas en un proyecto sin que se choquen entre ellas, ya que cada una trabaja en su rama, y cuando este lista actualizan el repositorio principal.
