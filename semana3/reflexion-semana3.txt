Cuando se empiezan a hacer proyectos de software complejos con otras personas, es fácil perder el rumbo de los cambios que has realizado y que archivos has modificado. Es debido a esta situación que existen herramientas como Git que junto con GitHub permiten versionar tu código, teniendo un historial de los diferentes cambios que se producen en el proyecto para poder identificar que cambios se hicieron; y en caso de que sea necesario revertir el código hecho para volver a una versión anterior, e incluso es posible crear diferentes ramas del proyecto para probar cambios sin modificar el repositorio principal. Además, Git es una herramienta fundamental para las empresas ya que facilita que el trabajo varias personas no choquen, ya que cada trabaja aparte, y actualiza cuando esté listo.
